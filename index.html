ICSD Student Chromebook Audit

Policy enforcement and bypass-vector assessment for managed student devices
What this tests: This tool focuses on things your content filter (Linewize) cannot tell you. Rather than testing whether specific URLs are blocked (Linewize handles that well), it checks Chrome policy enforcement, browser capability restrictions, bypass vectors that operate below the content-filter layer, and whether the filter extension itself is detectable. The findings identify gaps that content filtering alone cannot close.
Manual checks needed: Several important checks (developer mode, Powerwash, USB boot, chrome://policy, chrome://management) cannot be automated from JavaScript. Those are listed in the "Manual Verification" section at the bottom with step-by-step instructions. Run the automated tests first, then walk through the manual ones.

Device & Browser Fingerprint

Baseline info about the device under test. Used to document the testing environment and verify we're on a ChromeOS managed device.

Enterprise Management Indicators

Does the device report being managed? Checks for Linewize Connect extension and other indicators that enterprise policy is actually being enforced.

Chrome Policy Enforcement (Inferred)

Infers whether critical Chrome policies are enforced by testing browser behavior. We can't read policies directly from JS, but we can test whether the behaviors they control are locked down.

Browser Capabilities & Hardware APIs

Hardware-level APIs (USB, Serial, HID, Bluetooth) bypass any network filter entirely. If these are available to student JavaScript, a malicious webpage could talk directly to connected hardware.

WebRTC & Network Leakage

WebRTC can leak local network IPs, enable peer-to-peer tunneling (a known filter bypass), and typically is not caught by content filters. Chrome policy controls how WebRTC handles IPs.

Filter Bypass Methods

The bypass vectors that actually work in real K-12 environments. DNS-over-HTTPS is the big one - if students can enable secure DNS in Chrome, Linewize's DNS-level visibility is bypassed entirely.

Monitoring & Classroom Tools

Attempts to detect whether classroom monitoring tools (Classwize, GoGuardian, Securly, Hapara) are running. Tests whether student activity is being observed, not just filtered.

CIPA Sanity Check

Minimal spot-check to confirm Linewize is handling CIPA categories as expected. Not a comprehensive content filter test - just a sanity check on the most important categories.

Internal Infrastructure Reachability

Tests whether the student Chromebook can reach key internal resources from the browser. Complements the PowerShell network reachability tests with browser-layer visibility.

Manual Verification Checklist

These checks cannot be automated and require physically interacting with the device. Walk through each one and document the result.
