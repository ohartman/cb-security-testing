K-12 Chromebook Security Diagnostic

Policy enforcement and browser capability audit for managed student devices
Version 2.0 · Linewize-safe · Runs entirely in-browser · No data leaves this page
What this tool is: A read-only diagnostic that reports on Chrome enterprise policy enforcement state and browser capability exposure. It checks whether hardening controls are in place by observing browser behavior. Every finding comes with the specific policy name that controls it and guidance on how to fix it.
What this tool is not: This is not a penetration test. It does not attempt to bypass content filtering, exploit vulnerabilities, or demonstrate attacks. It cannot find zero-day bugs. It does not replace a professional security assessment. Its findings are a starting point for hardening, not a complete picture of your security posture.
For K-12 IT use: This tool is intended for use by school IT staff auditing their own devices on their own networks with proper authorization. Running it on devices you don't manage is not its intended use. The tool does not transmit any data off the device — all checks run in the browser and results stay local until you explicitly copy them out.

Device & Browser Fingerprint

Baseline info about the device under test. Purely local - no network requests.

Enterprise Management

Does the device report being managed, and by whom?

Chrome Policy Enforcement

Infers whether critical Chrome policies are enforced by testing observable behavior. We can't read policies directly, but we can test what they control.

Hardware APIs

Hardware-level APIs that can bypass network filters entirely. Each has a specific Chrome policy to control it.

Sensor & Device APIs

Sensor access APIs exposed to web pages. Can be used for fingerprinting or tracking.

Storage & Privacy APIs

Browser storage mechanisms and privacy-related APIs. Some enable legitimate functionality, some enable tracking.

Fingerprinting Surface

What a website can learn about the device for identification purposes. These are informational - the goal is to understand what's exposed, not to stop fingerprinting entirely (which is hard).

WebRTC & Network Leakage

WebRTC can leak local network IPs, enable peer-to-peer tunneling, and is typically not caught by content filters. Chrome policies control this behavior.

Network Information Disclosure

What the browser tells websites about your network connection.

Safe Reachability

Tests connectivity to services that should be allowed on a working student device.

Manual Verification Checklist

These checks cannot be automated from JavaScript and require physical interaction with the device. The most valuable findings often come from this section.
